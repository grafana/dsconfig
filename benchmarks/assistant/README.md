# Benchmarks — Grafana Assistant

> **Scope:** This directory covers **Grafana Assistant runs against the assistant's own
> `datasource-config` scenarios** — how well the assistant configures Grafana data sources through
> the `manage_datasources` tool. It is the assistant-flow counterpart to [`../mcp/`](../mcp/), which
> benchmarks the raw mcp-grafana tools. Don't mix the two.

## What's in here

| File | Purpose |
|---|---|
| `run.sh` | Orchestrator: brings up the local LLMSpec stack, runs the `datasource-config` scenarios, drops `report.html` + `latest.json` here, optionally commits + pushes. |
| `report.html` | Self-contained LLMSpec HTML report with per-scenario pass rates and full transcripts. **Generated by LLMSpec; do not edit by hand.** |
| `latest.json` | The run's structured job result (metrics, pass rate, cost, per-scenario list) — diffable across runs. **Generated.** |

Unlike [`../mcp/`](../mcp/), there is no `render.py`: LLMSpec's CLI writes the HTML report directly
via `--report`, so the whole harness is essentially the single `mise run llmspec` command below.

## Prerequisites

1. **The `grafana-assistant-app` repo checked out as a sibling** of this repo (i.e. `../../../grafana-assistant-app`
   relative to this folder). Override with `GA_APP_DIR=/path/to/grafana-assistant-app` if it lives elsewhere.
2. **Docker running** — LLMSpec spins up a Grafana + datasource stack with deterministic UIDs
   (`prometheus`, `loki`, `tempo`, `mysql`, `postgres`, `clickhouse`, `bigquery`, `influxdb`, `infinity`).
3. **`ANTHROPIC_API_KEY` exported** in your shell (used by the agent and the semantic grader).

## Usage

Run everything from this directory:

```bash
cd benchmarks/assistant

# Env up + run, writing report.html + latest.json here (no git changes)
./run.sh

# Same, but also commit + push the generated files to the current branch
PUBLISH=1 ./run.sh

# Reuse an already-running stack (skip the env bring-up)
SKIP_ENV=1 ./run.sh
```

Or, if the stack is already up, just the one command straight from the assistant repo:

```bash
mise run llmspec -- --agent=grafana_assistant --scenarios=assistant/datasource-config \
  --shots=3 --report=/path/to/benchmarks/assistant/report.html
```

### Environment variables

| Var | Default | Notes |
|---|---|---|
| `AGENT` | `grafana_assistant` | Which agent to benchmark (frontend Playwright runtime). |
| `SCENARIOS` | `assistant/datasource-config` | Scenario folder prefix (selects every `*.convo.yaml` under it). |
| `SHOTS` | `3` | Runs per scenario (N-shot / pass^N). |
| `MODEL` | _(LLMSpec default)_ | Agent model override (`--agent-model`), e.g. `claude-haiku-4-5-20251001`. |
| `SKIP_ENV` | `0` | `1` = don't run `llmspec:env:up`; reuse the current stack. |
| `PUBLISH` | `0` | `1` = commit + push the generated files after the run. |
| `GA_APP_DIR` | `../../../grafana-assistant-app` | Path to the grafana-assistant-app repo. |

## Viewing `report.html`

`report.html` is self-contained but GitHub serves raw `.html` as source, not rendered. To view it:

- **htmlpreview** (zero setup, public repos): prefix the file's GitHub URL with `https://htmlpreview.github.io/?`
- Or download and open locally.

The HTML report is the primary surface here: it lists each scenario's pass^N / pass@N and the full
tool-call transcripts (including the `manage_datasources` calls the scenarios assert on).
