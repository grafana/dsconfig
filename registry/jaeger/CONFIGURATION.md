## Trace to logs

_This section is optional._

### Trace to logs

_optional · object_

Navigate from a trace span to the selected data source's logs.

Navigate from a Jaeger trace span to the selected logs data source. Written by `@grafana/o11y-ds-frontend`'s `TraceToLogsSection` (v2 shape); the editor clears the legacy `tracesToLogs` key on any write.

Shape: `{ datasourceUid?: string, tags?: Array<{ key: string, value?: string }>, spanStartTimeShift?: string, spanEndTimeShift?: string, filterByTraceID?: boolean, filterBySpanID?: boolean, query?: string, customQuery: boolean }`.

- `datasourceUid` — target logs datasource UID (loki, elasticsearch, splunk, opensearch, falconlogscale, googlecloud-logging, victoriametrics-logs).
- `tags` — key/value tag mappings used to build the log query (default tags: `cluster`, `hostname`, `namespace`, `pod`, `service.name`, `service.namespace`).
- `spanStartTimeShift` / `spanEndTimeShift` — time-range shifts applied when jumping to logs (e.g. `-1m`, `2m`).
- `filterByTraceID` / `filterBySpanID` — filter the logs by the trace/span ID; disabled when `customQuery` is on.
- `customQuery` — use a custom query template; when true, `query` is used with `$__tags` interpolation.

Docs: [Trace to logs](https://grafana.com/docs/grafana/latest/datasources/jaeger/configure-jaeger-data-source/#trace-to-logs).

## Trace to metrics

_This section is optional._

### Trace to metrics

_optional · object_

Navigate from a trace span to the selected data source's metrics.

Navigate from a Jaeger trace span to the selected metrics data source (Prometheus or victoriametrics-metrics). Written by `@grafana/o11y-ds-frontend`'s `TraceToMetricsSection`.

Shape: `{ datasourceUid?: string, tags?: Array<{ key: string, value: string }>, queries: Array<{ name?: string, query?: string }>, spanStartTimeShift?: string, spanEndTimeShift?: string }`.

- `datasourceUid` — target metrics datasource UID.
- `tags` — key/value tag mappings used in the metrics query.
- `queries` — list of Prometheus queries offered on the trace→metrics link, each with a display name and a query template that interpolates `$__tags`.
- `spanStartTimeShift` / `spanEndTimeShift` — time-range shifts (default placeholders `-2m` / `2m`).

Docs: [Trace to metrics](https://grafana.com/docs/grafana/latest/datasources/jaeger/configure-jaeger-data-source/#trace-to-metrics).

### Span bar

_optional · object_

Add additional info next to the service and operation on a span bar row in the trace view.

Additional label rendered next to service/operation on each span bar row in the trace view. Written by `@grafana/o11y-ds-frontend`'s `SpanBarSection`.

Shape: `{ type?: 'None' | 'Duration' | 'Tag', tag?: string }`.

- `type` — label source. `'None'` hides the extra label; `'Duration'` (default, shown as placeholder in the editor) renders the span duration; `'Tag'` renders the value of the tag named by `tag`.
- `tag` — required when `type` is `'Tag'`; the tag key whose value is displayed (searched in span attributes and resource attributes).

Docs: [Span bar](https://grafana.com/docs/grafana/latest/datasources/jaeger/configure-jaeger-data-source/#span-bar).

### Query Trace by ID with Time Params

_optional · object_

pass time parameters when querying trace by ID.

Toggle whether time-range parameters (`start` / `end`) are appended when querying a trace by ID. Written by the plugin-local `TraceIdTimeParams` component (`src/configuration/TraceIdTimeParams.tsx`).

Shape: `{ enabled?: boolean }`. Default: disabled.

When `enabled` is `true` and the panel time range is present, the backend adds `start` and `end` query parameters to `GET /api/traces/{traceID}` (`pkg/jaeger/client.go:242-266`). Useful when Jaeger's storage backend requires a time hint to locate long-retained traces.
