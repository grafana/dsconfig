# Grafana Pyroscope configuration

Configuration reference for the **Grafana Pyroscope** data source (`grafana-pyroscope-datasource`) in Grafana.

For more information, see the [official documentation](https://grafana.com/docs/grafana/latest/datasources/pyroscope/).

> Generated from [`dsconfig.json`](dsconfig.json). Do not edit by hand — run `go generate ./...` to refresh.

## Fields

| Field | Type | Target | Required | Description |
|---|---|---|---|---|
| `url` | string | root | yes | URL |
| `basicAuth` | boolean | root |  |  |
| `basicAuthUser` | string | root | conditional | The username of the data source account |
| `secureJsonData.basicAuthPassword` 🔒 | string | secureJsonData | conditional | The password of the data source account |
| `jsonData.oauthPassThru` | boolean | jsonData |  |  |
| `jsonData.tlsAuthWithCACert` | boolean | jsonData |  | Add your own Certificate Authority (CA) certificate on top of one generated by the certificate authorities for additional security measures |
| `secureJsonData.tlsCACert` 🔒 | string (multiline) | secureJsonData | conditional | Your self-signed certificate |
| `jsonData.tlsAuth` | boolean | jsonData |  | Validate using TLS client authentication, in which the server authenticates the client |
| `jsonData.serverName` | string | jsonData | conditional | A Servername is used to verify the hostname on the returned certificate |
| `secureJsonData.tlsClientCert` 🔒 | string (multiline) | secureJsonData | conditional | The client certificate can be generated from a Certificate Authority or be self-signed |
| `secureJsonData.tlsClientKey` 🔒 | string (multiline) | secureJsonData | conditional | The client key can be generated from a Certificate Authority or be self-signed |
| `jsonData.tlsSkipVerify` | boolean | jsonData |  | Skipping TLS certificate validation is not recommended unless absolutely necessary or for testing |
| `jsonData.keepCookies` | list | jsonData |  | Grafana proxy deletes forwarded cookies by default. Specify cookies by name that should be forwarded to the data source. |
| `jsonData.timeout` | number | jsonData |  | HTTP request timeout in seconds |
| `jsonData.httpHeaders` | list | jsonData |  | Additional HTTP headers sent with every request. Header names are stored in jsonData (httpHeaderName<N>); header values are write-only in secureJsonData (httpHeaderValue<N>). |
| `jsonData.httpHeaders[].name` | string | jsonData | yes | Header |
| `jsonData.httpHeaders[].value` | string | jsonData |  | Value |
| `jsonData.minStep` | string | jsonData |  | Minimal step used for metric query. Should be the same or higher as the scrape interval setting in the Pyroscope database. |

## Provisioning examples

Each scenario below shows how to provision the data source in Grafana using a YAML file (loaded by Grafana's [file provisioner](https://grafana.com/docs/grafana/latest/administration/provisioning/#data-sources)) and using the [Grafana Terraform provider](https://registry.terraform.io/providers/grafana/grafana/latest/docs/resources/data_source).

Placeholders like `<YOUR_TOKEN>` must be replaced with real values before use.

### Default configuration

**Grafana provisioning YAML**

```yaml
apiVersion: 1
datasources:
  - name: Grafana Pyroscope
    type: grafana-pyroscope-datasource
    access: proxy
    basicAuth: false
    basicAuthUser: User
    url: "http://localhost:4040"
    jsonData:
      oauthPassThru: false
      serverName: domain.example.com
      tlsAuth: false
      tlsAuthWithCACert: false
      tlsSkipVerify: false
    secureJsonData:
      basicAuthPassword: "<YOUR_PASSWORD>"
      tlsCACert: "<YOUR_CA_CERTIFICATE>"
      tlsClientCert: "<YOUR_CLIENT_CERTIFICATE>"
      tlsClientKey: "<YOUR_CLIENT_KEY>"
```

**Terraform**

```hcl
resource "grafana_data_source" "grafana_pyroscope_datasource" {
  type = "grafana-pyroscope-datasource"
  name = "Grafana Pyroscope"
  url = "http://localhost:4040"

  json_data_encoded = jsonencode({
    oauthPassThru = false
    serverName = "domain.example.com"
    tlsAuth = false
    tlsAuthWithCACert = false
    tlsSkipVerify = false
  })

  secure_json_data_encoded = jsonencode({
    basicAuthPassword = "<YOUR_PASSWORD>"
    tlsCACert = "<YOUR_CA_CERTIFICATE>"
    tlsClientCert = "<YOUR_CLIENT_CERTIFICATE>"
    tlsClientKey = "<YOUR_CLIENT_KEY>"
  })
}
```
