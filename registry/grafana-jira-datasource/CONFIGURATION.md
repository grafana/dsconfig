# Jira configuration

Configuration reference for the **Jira** data source (`grafana-jira-datasource`) in Grafana.

For more information, see the [official documentation](https://grafana.com/docs/plugins/grafana-jira-datasource).

> Generated from [`dsconfig.json`](dsconfig.json). Do not edit by hand — run `go generate ./...` to refresh.

## Fields

| Field | Type | Target | Required | Description |
|---|---|---|---|---|
| `jsonData.hosting` | enum (cloud, server) | jsonData |  | The provider is where your JIRA instance is hosted. Select Jira Cloud if your Jira is being hosted on Atlassian Cloud. Select Jira Data Center / Jira Server if your Jira is being hosted elsewhere. |
| `jsonData.url` | string | jsonData | yes | The URL is the root URL for your Atlassian instance (example: company.atlassian.net) |
| `jsonData.authMethod` | enum (basicAuth, oauth2) | jsonData |  | Select the authentication method to use when connecting to Jira |
| `jsonData.user` | string | jsonData |  | The email address for the user/service account |
| `secureJsonData.token` 🔒 | string | secureJsonData | conditional | The API token is the token associated with the user. You can create one here. |
| `jsonData.scopedToken` | boolean | jsonData |  | Toggle this switch on if API Token with scopes is used |
| `jsonData.cloudId` | string | jsonData | conditional | Jira App Cloud Id is required to be used with API Token with Scopes |
| `jsonData.oauthClientID` | string | jsonData | conditional | The OAuth 2.0 Client ID for the Jira service account |
| `secureJsonData.oauthClientSecret` 🔒 | string | secureJsonData | conditional | The OAuth 2.0 Client Secret for the Jira service account |
| `jsonData.tlsAuthWithCACert` | boolean | jsonData |  | Add your own Certificate Authority (CA) certificate on top of one generated by the certificate authorities for additional security measures |
| `secureJsonData.tlsCACert` 🔒 | string (multiline) | secureJsonData | conditional | Your self-signed certificate |
| `jsonData.tlsAuth` | boolean | jsonData |  | Validate using TLS client authentication, in which the server authenticates the client |
| `jsonData.serverName` | string | jsonData |  | Server name used for TLS SNI / certificate verification. Optional; overrides the hostname used to verify the returned certificate. |
| `secureJsonData.tlsClientCert` 🔒 | string (multiline) | secureJsonData | conditional | The client certificate is encrypted and stored in the Grafana database. |
| `secureJsonData.tlsClientKey` 🔒 | string (multiline) | secureJsonData | conditional | The client key is encrypted and stored in the Grafana database. |
| `jsonData.tlsSkipVerify` | boolean | jsonData |  | Skipping TLS certificate validation is not recommended unless absolutely necessary or for testing |

## Provisioning examples

Each scenario below shows how to provision the data source in Grafana using a YAML file (loaded by Grafana's [file provisioner](https://grafana.com/docs/grafana/latest/administration/provisioning/#data-sources)) and using the [Grafana Terraform provider](https://registry.terraform.io/providers/grafana/grafana/latest/docs/resources/data_source).

Placeholders like `<YOUR_TOKEN>` must be replaced with real values before use.

### Basic Auth (`basicAuth`)

**Grafana provisioning YAML**

```yaml
apiVersion: 1
datasources:
  - name: Jira
    type: grafana-jira-datasource
    access: proxy
    jsonData:
      authMethod: basicAuth
      cloudId: Jira App Cloud Id
      hosting: cloud
      scopedToken: false
      tlsAuth: false
      tlsAuthWithCACert: false
      tlsSkipVerify: false
      url: URL
    secureJsonData:
      tlsCACert: "<YOUR_CA_CERT>"
      tlsClientCert: "<YOUR_CLIENT_CERT>"
      tlsClientKey: "<YOUR_CLIENT_KEY>"
      token: "<YOUR_API_TOKEN>"
```

**Terraform**

```hcl
resource "grafana_data_source" "grafana_jira_datasource_basicAuth" {
  type = "grafana-jira-datasource"
  name = "Jira"
  url = "https://example.com"

  json_data_encoded = jsonencode({
    authMethod = "basicAuth"
    cloudId = "Jira App Cloud Id"
    hosting = "cloud"
    scopedToken = false
    tlsAuth = false
    tlsAuthWithCACert = false
    tlsSkipVerify = false
    url = "URL"
  })

  secure_json_data_encoded = jsonencode({
    tlsCACert = "<YOUR_CA_CERT>"
    tlsClientCert = "<YOUR_CLIENT_CERT>"
    tlsClientKey = "<YOUR_CLIENT_KEY>"
    token = "<YOUR_API_TOKEN>"
  })
}
```

### OAuth 2.0 — Service Account (`oauth2`)

**Grafana provisioning YAML**

```yaml
apiVersion: 1
datasources:
  - name: Jira
    type: grafana-jira-datasource
    access: proxy
    jsonData:
      authMethod: oauth2
      cloudId: Jira App Cloud Id
      hosting: cloud
      oauthClientID: Client ID
      tlsAuth: false
      tlsAuthWithCACert: false
      tlsSkipVerify: false
      url: URL
    secureJsonData:
      oauthClientSecret: "<YOUR_CLIENT_SECRET>"
      tlsCACert: "<YOUR_CA_CERT>"
      tlsClientCert: "<YOUR_CLIENT_CERT>"
      tlsClientKey: "<YOUR_CLIENT_KEY>"
```

**Terraform**

```hcl
resource "grafana_data_source" "grafana_jira_datasource_oauth2" {
  type = "grafana-jira-datasource"
  name = "Jira"
  url = "https://example.com"

  json_data_encoded = jsonencode({
    authMethod = "oauth2"
    cloudId = "Jira App Cloud Id"
    hosting = "cloud"
    oauthClientID = "Client ID"
    tlsAuth = false
    tlsAuthWithCACert = false
    tlsSkipVerify = false
    url = "URL"
  })

  secure_json_data_encoded = jsonencode({
    oauthClientSecret = "<YOUR_CLIENT_SECRET>"
    tlsCACert = "<YOUR_CA_CERT>"
    tlsClientCert = "<YOUR_CLIENT_CERT>"
    tlsClientKey = "<YOUR_CLIENT_KEY>"
  })
}
```
