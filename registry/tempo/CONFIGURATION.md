## Streaming

### Streaming

_optional · object_

Enable streaming for different Tempo features.

Enable streaming for different Tempo features. If your Tempo instance is behind a load balancer or proxy that does not support gRPC or HTTP2, streaming will probably not work and should be disabled.

Shape: `{ search?: boolean, metrics?: boolean }`.

- `search` enables streaming for TraceQL search queries. Minimum required Tempo version: **2.2.0**.
- `metrics` enables streaming for TraceQL metrics queries. Minimum required Tempo version: **2.7.0**.

See [Stream query results](https://grafana.com/docs/tempo/latest/traceql/#stream-query-results) in the Tempo docs.

## Trace to logs

_This section is optional._

### Trace to logs

_optional · object_

Navigate from a trace span to the selected data source's logs.

Navigate from a trace span to the selected logs data source. Written by `@grafana/o11y-ds-frontend`'s `TraceToLogsSection` (v2 shape); the editor clears the legacy `tracesToLogs` key on any write.

Shape: `{ datasourceUid?: string, tags?: Array<{ key: string, value?: string }>, spanStartTimeShift?: string, spanEndTimeShift?: string, filterByTraceID?: boolean, filterBySpanID?: boolean, query?: string, customQuery: boolean }`.

- `datasourceUid` — target logs datasource UID (loki, elasticsearch, splunk, opensearch, falconlogscale, googlecloud-logging, victoriametrics-logs).
- `tags` — key/value tag mappings used to build the log query (default tags: `cluster`, `hostname`, `namespace`, `pod`, `service.name`, `service.namespace`).
- `spanStartTimeShift` / `spanEndTimeShift` — time-range shifts applied when jumping to logs (e.g. `-1m`, `2m`).
- `filterByTraceID` / `filterBySpanID` — filter the logs by the trace/span ID; disabled when `customQuery` is on.
- `customQuery` — use a custom query template; when true, `query` is used with `$__tags` interpolation.

Docs: [Trace to logs](https://grafana.com/docs/grafana/latest/datasources/tempo/configure-tempo-data-source/#trace-to-logs).

### Trace to metrics

_optional · object_

Navigate from a trace span to the selected data source's metrics.

Navigate from a trace span to the selected metrics data source (Prometheus or victoriametrics-metrics). Written by `@grafana/o11y-ds-frontend`'s `TraceToMetricsSection`.

Shape: `{ datasourceUid?: string, tags?: Array<{ key: string, value: string }>, queries: Array<{ name?: string, query?: string }>, spanStartTimeShift?: string, spanEndTimeShift?: string }`.

- `datasourceUid` — target metrics datasource UID.

- `queries` — list of Prometheus queries offered on the trace→metrics link, each with a display name and a query template that interpolates `$__tags`.
- `spanStartTimeShift` / `spanEndTimeShift` — time-range shifts (default placeholders `-2m` / `2m`).

Docs: [Trace to metrics](https://grafana.com/docs/grafana/latest/datasources/tempo/configure-tempo-data-source/#trace-to-metrics).

_optional · object_

Modify how traces are searched.

Configure the TraceQL search tab in the query editor.

Shape: `{ hide?: boolean, filters?: TraceqlFilter[] }`.

- `hide` — remove the search tab from the query editor (tooltip: "Removes the search tab from the query editor").
- `filters` — pre-configured static filters exposed in the UI. Each filter is `{ id: string, tag?: string, operator?: string, scope?: 'event'|'instrumentation'|'intrinsic'|'link'|'resource'|'span'|'unscoped', value?: string | string[], valueType?: string, isCustomValue?: boolean }`.

## TraceID query

_This section is optional._

### TraceID query

_optional · object_

Modify how TraceID queries are run.

Control the time range applied when running a query by TraceID.

Shape: `{ timeShiftEnabled?: boolean, spanStartTimeShift?: string, spanEndTimeShift?: string }`.

- `timeShiftEnabled` — "Use time range in query" toggle (tooltip: "The time range can be used when there are performance issues or timeouts since it will narrow down the search to the defined range. Default: disabled"). Default: `false`.
- `spanStartTimeShift` — "Time shift for start of search" (tooltip: "Shifts the start of the time range when searching by TraceID. Searching can return traces that do not fully fall into the search time range, so we recommend using higher time shifts for longer traces. Default: 30m (Time units can be used here, for example: 5s, 1m, 3h").
- `spanEndTimeShift` — "Time shift for end of search" (same rule for the end of the range).

Inputs are disabled unless `timeShiftEnabled` is `true`.

Docs: [TraceID query](https://grafana.com/docs/grafana/latest/datasources/tempo/configure-tempo-data-source/#traceid-query).
